Internet & Network Overview

🌐

What is the Internet?

A global "network of networks" — interconnected systems communicating over TCP/IP. Originated from ARPANET (1969), funded by the US Department of Defense. The first long-distance communication linked MIT to California in 1965.

📡

Data Communication

Exchange of data between two devices via a transmission medium. Governed by protocols — rules defining syntax (structure), semantics (meaning), and timing (when/how fast) of data exchange.

🔗

Network Node

Any device capable of sending/receiving data: computers, routers, switches, printers. Nodes communicate via peer-to-peer interfaces or service interfaces across layered protocol stacks.

🏗️

Network Infrastructure

Access Networks connect end-systems to the first router. Core Networks interconnect routers and ISPs using high-bandwidth links (1.5 Mbps – 622 Mbps+). ISP hierarchy: local → regional → national (NBPs).

🔐 Cybersecurity Relevance

Understanding network infrastructure is the foundation of attack surface analysis. Every access network is a potential entry point; every protocol has vulnerabilities. Know the terrain before you defend (or test) it.

Network Types by Geography

LAN

Local

High-speed, short-distance (meters to ~1km). Single building or campus. Privately owned. Ethernet, Wi-Fi. Attack surface: ARP spoofing, VLAN hopping, rogue AP

MAN

Metro

City-wide extension of LAN. Spans multiple LANs over a city. Can share resources across departments or branches.

WAN

Wide

Collection of networks (LANs) across countries/continents. Provider-managed. Slower than LAN. Attack surface: BGP hijacking, MPLS vulnerabilities

⚠️ Why -2 for Hosts?

Network address (all host bits = 0) and broadcast address (all host bits = 1) are reserved. A /24 subnet has 256 addresses but only 254 usable hosts.

Subnetting Concepts

Subnetting divides a network into smaller logical segments using a subnet mask. It improves security by isolating segments and reduces broadcast traffic.

Subnet Mask Logic

A subnet mask is a 32-bit number. 1s = network portion, 0s = host portion.

IP:   192.168.1.100
Mask: 255.255.255.0  (/24)

Binary:
IP:   11000000.10101000.00000001.01100100
Mask: 11111111.11111111.11111111.00000000
Net:  11000000.10101000.00000001.00000000
      = 192.168.1.0

CIDR Notation

Classless Inter-Domain Routing (CIDR) uses prefix length after a slash.

/8  = 255.0.0.0    (16M hosts)
/16 = 255.255.0.0  (65K hosts)
/24 = 255.255.255.0  (254 hosts)
/25 = 255.255.255.128 (126 hosts)
/30 = 255.255.255.252 (2 hosts)
/32 = 255.255.255.255 (1 host)
🔐 Security Use of Subnetting

Network segmentation is a core defense strategy. DMZ (Demilitarized Zone) uses subnets to isolate public-facing servers from the internal network. VLAN implementation maps to subnet architecture.

IPv6 — Next Generation Addressing

Key Differences from IPv4

  • Address size: 128 bits (vs 32 bit) = 3.4 × 10³⁸ addresses
  • Notation: Eight 16-bit hex groups separated by colons
    2001:0db8:85a3::8a2e:0370:7334
  • No broadcasts: Uses multicast instead
  • Built-in IPSec: Security headers are native
  • No NAT needed: Enough addresses for every device
  • Auto-configuration: SLAAC (Stateless Address Auto-config)

IPv6 Address Types

  • Unicast: Global (2000::/3), Link-local (fe80::/10), Loopback (::1)
  • Multicast: Prefix ff00::/8 — replaces broadcast
  • Anycast: One address, multiple interfaces — nearest responds
⚠️ IPv6 Security Risks

Many firewalls are IPv4-only. Attackers use IPv6 tunnels to bypass. Neighbor Discovery Protocol (NDP) lacks authentication — vulnerable to ND Spoofing.

⚔️ SYN Flood Attack

Attacker sends thousands of SYN packets with spoofed source IPs. Server enters SYN_RECEIVED for each, consuming connection table (backlog) until legitimate connections are refused. Defense: SYN cookies, rate limiting, firewall rules.

TCP Flags Reference
SYN=Synchronize | ACK=Acknowledge | FIN=Finish
RST=Reset | PSH=Push | URG=Urgent

TCP Header Fields

  • Source/Destination Port: 16-bit each
  • Sequence Number: 32-bit, byte ordering
  • Acknowledgment Number: Next expected byte
  • Window Size: Flow control buffer size
  • Checksum: Error detection
  • Urgent Pointer: For URG flag data

TCP vs UDP at a Glance

  • TCP: Connection-oriented, reliable, ordered, slower
  • UDP: Connectionless, unreliable, no ordering, faster
  • TCP uses: HTTP, FTP, SSH, SMTP, POP3
  • UDP uses: DNS, DHCP, VoIP, video streaming, SNMP

UDP — User Datagram Protocol

Characteristics

  • Connectionless: No handshake — fire and forget
  • Unreliable: No guarantee of delivery or order
  • Fast: Minimal overhead (8-byte header vs TCP's 20-byte)
  • Broadcast/Multicast: Supports one-to-many
UDP Header (8 bytes):
[Src Port 16] [Dst Port 16]
[Length  16] [Checksum 16]

UDP Attacks

UDP Flood DDoS
HIGH

Overwhelms target with UDP packets to random ports, forcing ICMP "port unreachable" responses. Exhausts bandwidth/CPU.

DNS Amplification
CRITICAL

Exploits open DNS resolvers via UDP. Attacker spoofs victim IP, resolver sends large responses (100x amplification) to victim.

DNS — Domain Name System

How DNS Works

  1. Browser checks local cache
  2. Queries OS resolver (checks /etc/hosts)
  3. Queries Recursive DNS resolver (ISP or 8.8.8.8)
  4. Resolver queries Root Name Server (13 roots)
  5. Root directs to TLD Name Server (.com, .org)
  6. TLD directs to Authoritative NS
  7. IP returned, cached with TTL

DNS Cache Poisoning:

ARP — Address Resolution Protocol

How ARP Works

ARP maps IP addresses to MAC addresses on a local network. Uses a broadcast mechanism.

Who has 192.168.1.1? (Broadcast)
→ All devices receive it
→ 192.168.1.1 responds:
  "I'm at AA:BB:CC:DD:EE:FF"
→ Stored in ARP cache

DHCP — Dynamic Host Config

Automatically assigns IP configuration using DORA process:

  • Discover — Client broadcasts
  • Offer — Server offers IP
  • Request — Client requests offered IP
  • Acknowledge — Server confirms lease
⚔️ ARP Poisoning Attack

Attacker sends gratuitous ARP replies associating their MAC with another host's IP. ARP caches are poisoned — traffic destined for the victim flows to the attacker (Man-in-the-Middle). Defense: Dynamic ARP Inspection (DAI), static ARP entries, network monitoring with arpwatch.

Circuit Switching

Dedicated path reserved for entire session. Phases: Setup → Data Transfer → Teardown. Example: PSTN telephone networks.

  • + Guaranteed bandwidth
  • + Low, predictable delay
  • − Wastes bandwidth when idle
  • − Setup overhead

Packet Switching (Datagram)

Messages split into packets. Each packet routed independently using routing tables. Used by the Internet (IP).

  • + Efficient resource use
  • + Fault tolerant (rerouting)
  • − Variable delay (jitter)
  • − Out-of-order delivery possible

Virtual Circuit Switching

Hybrid: path set up in advance (like circuit), data sent in packets. VCI (Virtual Circuit ID) changes per hop. Example: ATM, Frame Relay, MPLS.

  • + Ordered delivery guaranteed
  • + QoS control
  • − Setup delay
  • − Complex state management

🔐 TTL as a Fingerprinting Tool

Default TTL reveals OS: Linux=64, Windows=128, Cisco=255. By counting hops (128-TTL seen), you can estimate network distance. Used in passive OS fingerprinting by Nmap, p0f, and Wireshark.

Network Security Fundamentals

Firewall Types

Packet Filter (L3/L4)

Inspects IP/TCP/UDP headers. Rules based on source IP, destination IP, port, protocol. Stateless — each packet evaluated independently.

⚠️ Cannot detect application-layer attacks. Easily evaded by fragmentation.

Stateful Inspection (L4)

Tracks connection state (SYN→ESTABLISHED). Knows if a packet belongs to an existing connection. Blocks unexpected packets.

✅ Prevents TCP state hijacking. Standard in modern firewalls.

Application Gateway / Proxy (L7)

Understands specific protocols (HTTP, FTP). Acts as relay — terminates and re-initiates connections. Deep packet inspection (DPI).

🔐 Can detect SQLi, XSS, protocol abuse. Used in WAF (Web Application Firewall).

Firewall Rule Logic

# Example iptables rules (Linux firewall)
iptables -A INPUT -p tcp --dport 22 -s 10.0.0.0/24 -j ACCEPT  # Allow SSH from LAN
iptables -A INPUT -p tcp --dport 22 -j DROP                    # Block SSH from everywhere else
iptables -A INPUT -p tcp --dport 80 -j ACCEPT                  # Allow HTTP
iptables -A INPUT -p tcp --dport 443 -j ACCEPT                 # Allow HTTPS
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT  # Allow return traffic
iptables -A INPUT -j DROP                                       # Default deny

NAT — Network Address Translation

How NAT Works

NAT replaces private source IPs with a single public IP. The router maintains a translation table mapping {private IP, private port} → {public IP, public port}.

Private:  192.168.1.100:54321
   ↓ NAT Router ↓
Public:   203.0.113.5:1024

Translation Table:
192.168.1.100:54321 ↔ 203.0.113.5:1024
192.168.1.101:54322 ↔ 203.0.113.5:1025

NAT Types

  • Static NAT: 1:1 mapping, public IP per private host
  • Dynamic NAT: Pool of public IPs, assigned on demand
  • PAT / NAT Overload: Many-to-one using ports (most common — home routers)

🔐 NAT provides implicit security — external hosts cannot initiate connections to internal hosts (no direct path). However, it breaks end-to-end connectivity and complicates some protocols (FTP, SIP).

IPSec Modes

  • Transport Mode: Encrypts only the payload. Original IP header preserved. Used for host-to-host.
  • Tunnel Mode: Entire packet (including original header) encrypted and wrapped in a new packet. Used for site-to-site VPNs.

⚠️ VPN ≠ Anonymous. Logs may exist. DNS leaks are common. Split tunneling can bypass VPN for some traffic.

Encryption & Digital Signatures

Symmetric Encryption

Same key for encryption and decryption. Fast, used for bulk data.

Plaintext
⬇ AES-256 + Key
Ciphertext: 3f8a2b...
⬇ AES-256 + Same Key
Plaintext
  • AES: Current standard (128/192/256-bit keys)
  • DES/3DES: Deprecated, weak
  • ChaCha20: Used in TLS 1.3, mobile

Asymmetric Encryption

Public key encrypts; private key decrypts. Slower, used for key exchange and signatures.

  • RSA: 2048/4096-bit. Based on prime factoring
  • ECC: Elliptic curve — shorter keys, same strength
  • Diffie-Hellman: Key exchange protocol
  • ECDH: Elliptic curve DH — used in TLS 1.3

🔐 Digital Signature: Sender signs with private key → recipient verifies with sender's public key. Provides authentication + non-repudiation.

IDS / IPS — Intrusion Detection/Prevention

Detection Methods

  • Signature-based: Matches known attack patterns. Fast, no false positives for known threats. Cannot detect 0-days.
  • Anomaly-based: Establishes baseline, flags deviations. Catches novel attacks but higher false positive rate.
  • Heuristic: Rules derived from attack behaviors, not exact signatures.
  • Behavioral: ML-based, learns normal user/system behavior.

Placement Strategies

  • NIDS: Network IDS — monitors all traffic on a segment (promiscuous mode / SPAN port)
  • HIDS: Host IDS — monitors a single system (logs, file integrity, syscalls)
  • IPS: Inline — can block traffic in real time (but adds latency)
  • Honeypot: Decoy system that attracts attackers, capturing TTPs

Network Attack Taxonomy

Study Note

Understanding how attacks work is essential for defense. This section maps attacks to OSI layers — helping you understand which controls apply at each layer.

Man-in-the-Middle (MitM)
Layer 2-3 | ARP/DNS Spoofing
CRITICAL

Attacker intercepts communication between two parties. Achieved via ARP poisoning, DNS hijacking, SSL stripping, rogue Wi-Fi. Defense: Certificate pinning, HSTS, mutual TLS, DNSSEC.

DDoS — Distributed Denial of Service
Layer 3-7 | Volumetric / Protocol / Application
CRITICAL

Flood target with traffic from many sources (botnets). Types: volumetric (bandwidth), protocol (SYN flood), application (HTTP flood). Defense: CDN, rate limiting, anycast, scrubbing centers.

Port Scanning
Layer 4 | Reconnaissance
MEDIUM

Nmap, masscan used to discover open ports and services. SYN scan (half-open) is stealthiest. Defense: Firewall rules, port knocking, network monitoring.

BGP Hijacking
Layer 3 | Routing Protocol
CRITICAL

Malicious AS announces more-specific prefixes, diverting internet traffic. Major incidents: YouTube 2008, Amazon Route53 2018. Defense: RPKI, BGPsec, route filtering.

SSL Stripping
Layer 7 | TLS Downgrade
HIGH

Attacker downgrades HTTPS connections to HTTP (often via MitM). Victim's browser sees HTTP. Defense: HSTS (HTTP Strict Transport Security), preloading HSTS.

VLAN Hopping
Layer 2 | Switch Attacks
HIGH

Switch Spoofing or Double Tagging allows attacker to access traffic in different VLANs. Defense: Disable DTP, use dedicated native VLAN, prune VLANs.

Packet Sniffing
Layer 1-2 | Passive
HIGH

Capturing network traffic with tools like Wireshark, tcpdump. Effective on shared media or after ARP poisoning. Defense: Encryption (TLS), switched networks, network segmentation.

IP Spoofing
Layer 3 | Source Forgery
HIGH

Forging source IP in packets. Used to bypass IP-based ACLs, amplification DDoS, blind TCP injection. Defense: BCP38 ingress filtering (ISP level), stateful firewalls.
